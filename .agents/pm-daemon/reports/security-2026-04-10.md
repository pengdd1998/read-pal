## Findings by Severity

### P0 (Critical) - Exploitable Now

#### P0-1: Weak Default JWT Secret in Production
**Location**: `/Volumes/ExternalDisk/read-pal/packages/api/src/middleware/auth.ts:57`
```typescript
const secret = process.env.JWT_SECRET || 'dev-secret-change-in-production';
```
**Issue**: Hardcoded fallback JWT secret that could be used in production if `JWT_SECRET` environment variable is not set.

**Impact**: 
- Attackers can forge JWT tokens
- Complete account takeover of any user
- Unauthorized access to all API endpoints

**Recommendation**:
```typescript
const secret = process.env.JWT_SECRET;
if (\!secret) {
  throw new Error('JWT_SECRET environment variable must be set in production');
}
```
**Evidence**: This pattern appears twice in `auth.ts` (lines 57 and 128).

---

#### P0-2: Missing Authentication on Health Check Endpoints
**Location**: `/Volumes/ExternalDisk/read-pal/packages/api/src/routes/health.routes.ts:22,39`
```typescript
router.get('/', async (_req, res) => { ... }); // No auth middleware
router.get('/detailed', async (_req, res) => { ... }); // No auth middleware
```
**Issue**: Health check endpoints expose detailed service status, configuration, and error messages without authentication.

**Impact**:
- Information disclosure about service architecture
- Exposure of database connection status
- Potential for service discovery attacks
- API version and environment disclosure

**Recommendation**:
- Either add authentication middleware for production environments
- Or implement IP whitelisting for health endpoints
- Consider using a separate monitoring port (common pattern: port 8080 for app, 9090 for metrics)

**Alternative**: If these must be public, ensure they don't expose sensitive information.

### P1 (Important) - Should Fix Soon

#### P1-1: DOMPurify Not Configured for All Use Cases
**Locations**: 
- `/packages/web/src/components/reading/CompanionChat.tsx:311`
- `/packages/web/src/components/reading/ReaderView.tsx:223`
- `/packages/web/src/app/chat/page.tsx:248`

**Finding**: ✅ **GOOD** - All uses of `dangerouslySetInnerHTML` properly use `DOMPurify.sanitize()`

**Current Implementation**:
```typescript
dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(renderSimpleMarkdown(msg.content)) }}
```

**Minor Hardening Opportunity**:
- Consider adding custom DOMPurify configuration to allow specific tags only
- Example: Add `data-*` attributes support for custom styling
- Current usage is safe but permissive

**Recommendation**:
```typescript
DOMPurify.addHook('uponSanitizeAttribute', (node, data) => {
  // Allow specific data attributes
  if (data.attrName.startsWith('data-')) {
    data.forceKeepAttr = true;
  }
});
```

#### P1-2: Public Route Exposes Free Book Catalog Without Rate Limiting
**Location**: `/packages/api/src/routes/discovery.routes.ts:135-191`

**Issue**: `GET /api/discovery/free-books` has authentication but no aggressive rate limiting.

**Current State**: Has `authenticate` middleware (line 135), which is good.

**Recommendation**: Add specific rate limiting for catalog endpoints to prevent scraping:
```typescript
router.get('/free-books', 
  authenticate,
  rateLimiter({ windowMs: 60000, max: 30 }), // Add this
  async (req: AuthRequest, res) => { ... }
);
```

## Positive Security Findings

### ✅ Proper SQL Injection Prevention
All database queries use proper parameterization:
- Sequelize ORM with proper `where` clauses
- No raw SQL with user input concatenation
- Template literals use bound parameters (e.g., `sequelize.query` with `{ bind: [userId] }`)

**Example from stats.routes.ts:165**:
```typescript
sequelize.query(
  `SELECT ... WHERE "userId" = $1 ...`,
  { bind: [userId], type: QueryTypes.SELECT }
)
```

### ✅ Comprehensive Authentication Coverage
All sensitive routes properly use `authenticate` middleware:
- `/api/auth/*` - Login/register with rate limiting
- `/api/books/*` - Book CRUD operations
- `/api/agents/*` - AI agent interactions
- `/api/annotations/*` - User annotations
- `/api/settings/*` - User preferences
- `/api/upload/*` - File uploads
- `/api/friend/*` - Friend agent
- `/api/stats/*` - User statistics

### ✅ Proper Password Handling
- Uses bcrypt with 12 rounds (BCRYPT_ROUNDS = 12)
- Enforces max password length (72 chars for bcrypt)
- Rate limiting on auth endpoints (10 req/min for login, 5 req/min for register)

### ✅ Input Validation
- express-validator for request body validation
- Length checks on string inputs
- Type checking for numeric values
- Enum validation for status fields

### ✅ CORS and Security Headers
(Need to verify in production configuration - code shows proper structure)

### ✅ File Upload Security
From `upload.routes.ts`:
- File type validation by MIME type and extension
- File size limits (50MB max)
- Filename sanitization to prevent path traversal
- Uses temp files with cleanup
