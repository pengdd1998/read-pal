# Alex — Tech Professional

## Profile
- **Age:** 30, senior software engineer
- **Reading habits:** 4-5 technical books/year, lots of documentation and articles
- **Tech comfort:** Very high — early adopter, uses AI tools daily
- **Motivation:** Learning new technologies, staying current, efficiency
- **Patience:** Low — expects polished UX, fast performance, keyboard shortcuts

## Simulation Tasks
1. **Speed Test**: Time from login to reading. Expect < 10 seconds.
2. **Technical Reading**: Upload a programming book. Test code block rendering, diagram handling.
3. **AI Deep Dive**: Ask the AI to "explain this algorithm like I'm implementing it". Test technical accuracy.
4. **Cross-Reference**: Upload 2 tech books. Ask "compare how these authors approach [topic]".
5. **Performance**: Check page load times, chat response latency, scroll smoothness.

## Success Criteria (from Alex's perspective)
- The app is fast — no noticeable lag anywhere
- AI understands technical content and gives precise answers
- Code blocks and technical formatting render correctly
- Keyboard shortcuts for common actions (highlight, note, search)
- The architecture is interesting enough to look at the codebase

## Failure Signals
- Slow load times or laggy scrolling
- AI gives incorrect or superficial technical answers
- Code formatting breaks in EPUB rendering
- No keyboard shortcuts or power-user features
- WebSocket disconnections during chat
